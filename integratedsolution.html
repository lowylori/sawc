Reporting System

The integrated reporting system uses Microsoft Power Automate to create a system of tools that would create an HTML document of the report, upload this to OneDrive, create a PDF out of that report, upload the PDF to OneDrive, and finally email a link with the PDF to anyone on the email list. This would run on a timed basis, depending on when SAWC wants the reports created and sent out. In practice, there would be three flows, one for each of the levels of priority, all that go out at different times. Below is a screenshot of the overall flow.

  1. Recurrance

    2. The recurrence happens whenever the report should be sent out. This flow is sent out once a month.

  2. List Rows Present in Table

    3. The excel sheet previously created is access, and is later called in dynamic content.

  3. Initialize Variable

    4. This creates a counter starting at 0. This is later used to count through how many incidents have occured in the month.

  4. Initialize Variable 2

    5. Creates the date variable, sets it to the current date with the format mm/dd/yyyy, or just mm if its daily

  5. For Each

    6. For each is the equivalent of a for loop. The loop iterates through the excel table.

  6. If the days/months in the rows are equivalent to the current date, the counter is incremented by 1.
  7. HTML to Text

    8. This action is where HTML is applied to create a report. Right now, there is only the title of the report and how many reports have occured. Input from SAWC would be needed to understand what else is required in the report.

  8. Create File

    9. The HTML file is uploaded to OneDrive. The date is used in the name so that the file does not overwrite another file.

  9. Convert File Using Path

    10. The HTML file is converted to a PDF, also with the date in the name.

  10. Create File 1

    11. The newly created PDF is uploaded to OneDrive.

  11. Sent an Email

    12. An email is sent with a link to the PDF.
